JAVASCRIPT RULES — NON-NEGOTIABLE 
 
Rule 1 — ASCII numerics only: All numeric values in JavaScript arrays and arithme9c 
expressions must use ASCII characters exclusively. The Unicode minus sign (U+2212) is not a 
valid JavaScript operator and will cause a syntax error. Use only the ASCII hyphen-minus

Rule 2 — Chart type data constraints: Doughnut, pie, and radar charts must receive only 
posi9ve numeric values. If a value is nega9ve, express it as an absolute value and adjust the 
label to communicate direc9on. 
 
Rule 3 — Independent chart guards: Each canvas element in a build func9on must have its

mul9ple charts. The correct palern is:

if (ctx) { new Chart(ctx, ...) } 
 
Rule 4 — ID-based tab switching: Tab panes must be shown and hidden using 
getElementById('tab-N') keyed directly to the bulon index. Never use querySelectorAll('.tab-
pane') with index matching. 
 
Rule 5 — Scroll to top: The showTab func9on must call window.scrollTo({ top: 0, behavior: 
'smooth' }) as its last line. 
 
Rule 6 — Post-genera9on silent valida9on: Before considering the dashboard complete, 
internally verify: (a) no Unicode minus in any JS data array, (b) every canvas id referenced in 
JavaScript has a matching canvas element in HTML, (c) every CSS variable used in HTML or 
inline styles is declared in :root, (d) no doughnut, pie, or radar chart receives nega9ve data 
values, (e) the showTab func9on uses getElementById not querySelectorAll. Fix any
