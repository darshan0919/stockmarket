# Local Jira-style issue docs

Markdown files in this folder are **repository-local** work items: same information you would put on a Jira ticket (status, priority, summary, root cause, fix, verification), without requiring access to an external Jira instance.

When the user says **“log Jira”** (or similar), add or update a doc here using the **Tracking fields** table pattern in existing issues. Optionally fill **Issue Key** if the work is mirrored in real Jira.

See project rule: `.cursor/rules/jira.mdc`.
